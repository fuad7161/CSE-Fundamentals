# Normalization: 1NF to BCNF

Explains the process of organizing a relational database to reduce redundancy and avoid anomalies.

- **1NF (First Normal Form):** atomic columns, no repeating groups
- **2NF (Second Normal Form):** 1NF + no partial dependency on a composite key
- **3NF (Third Normal Form):** 2NF + no transitive dependency on a non-key attribute
- **BCNF (Boyce-Codd Normal Form):** stricter version of 3NF (every determinant is a candidate key)
- Update, insertion, and deletion anomalies
- Functional dependencies and Armstrong's axioms
- Practical normalization examples and denormalization trade-offs
