# Pipelining & DLX pipeline stages

Introduction to instruction pipelining and the classic DLX five-stage pipeline.

- Concept of pipelining: overlapping instruction execution to improve throughput
- DLX five-stage pipeline:
  - **IF** — Instruction Fetch
  - **ID** — Instruction Decode / Register Read
  - **EX** — Execute / ALU operation
  - **MEM** — Memory access
  - **WB** — Write Back
- Pipeline diagrams and timing overview
- Speedup ideal vs. real (due to hazards and stalls)
