# Purpose of SSL/TLS Certificates & Certificate Authorities

How trust is established in secure web communications. Covers:

- **SSL/TLS certificate** — digital document binding a public key to an entity
- **Certificate Authority (CA)** — trusted third party that issues and signs certificates
- Chain of trust: root CA → intermediate CA → server certificate
- Certificate validation: browser checks signature, expiry, revocation (CRL/OCSP)
- Types: DV (Domain Validation), OV (Organization Validation), EV (Extended Validation)
- Let's Encrypt — automated, free certificate issuance
