# Software Design — coupling and cohesion

Key software design principles focusing on module interdependence and focus.

- **Coupling**: degree of interdependence between modules
  - Low coupling (loose coupling) is desirable
  - Types: data, stamp, control, common, content (worst)
- **Cohesion**: degree to which elements inside a module belong together
  - High cohesion (strong cohesion) is desirable
  - Types: coincidental, logical, temporal, procedural, communicational, sequential, functional (best)
- Relationship between coupling and cohesion
- Design guidelines: strive for high cohesion and low coupling
- Impact on maintainability, reusability, and testability
