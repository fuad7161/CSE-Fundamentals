# Networking Basics

Fundamental networking concepts that are part of the computer fundamentals syllabus — network types, internet basics, and essential network devices.

### 13.1 — LAN, WAN, MAN — Definitions & Differences
- LAN (Local Area Network): small area (office, building), high speed, privately owned
- MAN (Metropolitan Area Network): city-wide, medium speed
- WAN (Wide Area Network): large geographic area (countries), relatively slower, public/private (internet is the largest WAN)
- PAN (Personal Area Network): very short range (Bluetooth, ~10m)

### 13.2 — Internet vs Intranet vs Extranet
- Internet: global public network
- Intranet: private network within an organization (uses internet tech)
- Extranet: intranet extended to selected external parties (suppliers, partners)

### 13.3 — IP Address, Domain Name, URL
- IP Address: unique identifier for a device on a network (IPv4: 32-bit, IPv6: 128-bit)
- Domain Name: human-readable name (google.com) mapped to IP via DNS
- URL (Uniform Resource Locator): full web address (protocol + domain + path)
- DNS resolution: domain name → IP address

### 13.4 — Network Devices — Router, Switch, Hub, Modem
- Hub: broadcasts data to all ports (dumb, layer 1)
- Switch: sends data only to intended device (intelligent, layer 2)
- Router: connects different networks, routes data (layer 3), uses IP addresses
- Modem: modulates/demodulates signals (analog ↔ digital), connects to ISP
- Access Point: provides wireless connectivity
- Gateway: translates between different network protocols

## Supplementary Notes from Master Study Guide

### Modem
- **Modulator:** Converts digital to analog

### Types of Networks (Detailed)

|---|---|---|---|
| LAN | Building/Campus | High | Office network |
| MAN | City | Moderate | Cable TV |
| WAN | Country/Continent | Low | Internet |
| Internet | Global | Variable | Worldwide web |

### Network Architectures

| Architecture | Description | Advantages | Disadvantages |
|---|---|---|---|
| Client-Server | Centralized server | Secure, manageable | Single point of failure |
| Peer-to-Peer | Decentralized | Simple, cheap | Less secure |
| VAN | Value-added services | Extra features | Costly |

### Network Topologies

| Topology | Pros | Cons |
|---|---|---|
| Bus | Less cabling, simple | Single point of failure |
| Star | Easy error detection | Hub failure affects all |
| Ring | Equal access | One failure affects all |
| Mesh | Reliable, multiple paths | Very expensive |
| Tree | Hierarchical organization | Root failure affects all |

|---|---|

### OSI Model (7 Layers)

| Layer | Function |
|---|---|
| 7. Application | User interface |
| 6. Presentation | Data formatting, encryption |
| 5. Session | Session management |
| 4. Transport | End-to-end delivery |
| 3. Network | Routing |
| 2. Data Link | Error detection/correction |
