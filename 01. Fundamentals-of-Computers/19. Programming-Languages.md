# Programming Languages

Study notes covering the history, generations, translators, paradigms, and characteristics of programming languages.

---

## History of Programming Languages

| Period | Language | Characteristics |
|---|---|---|
| 1843 | Analytical Engine | First program by **Ada Lovelace** (first programmer) |
| 1940s | Machine Language | Binary code (0s and 1s), machine-dependent |
| 1950s | Assembly Language | Mnemonics (MOV, ADD, SUB), machine-dependent |
| 1957 | FORTRAN | First high-level language, scientific computing |
| 1958 | LISP | Functional programming, AI research |
| 1959 | COBOL | Business-oriented, English-like syntax |
| 1964 | BASIC | General-purpose, simple syntax for beginners |
| 1970 | PASCAL | Procedural, structured programming, teaching |
| 1972 | C | System software, powerful, portable (Dennis Ritchie) |
| 1979 | C++ | Object-oriented, enhanced C (Bjarne Stroustrup) |
| 1995 | Java | Platform-independent, web applications (James Gosling, Sun Microsystems) |

---

## Generations of Languages

| Generation | Type | Examples | Features |
|---|---|---|---|
| **1GL** | Machine Language | Binary code | Machine-dependent, fastest execution |
| **2GL** | Assembly Language | Mnemonics | Machine-dependent, low-level |
| **3GL** | High-Level Language | FORTRAN, C, COBOL, Java | Machine-independent, easier to write/read |
| **4GL** | Very High-Level | SQL, PowerBuilder | Database-oriented, less code, declarative |
| **5GL** | AI Languages | Prolog, Mercury | Constraint-based, logic programming |

---

## Language Translators

| Translator | Process | Output |
|---|---|---|
| **Compiler** | Translates **entire program** at once | Machine code (executable file) |
| **Interpreter** | Translates and executes **line by line** | Machine code (no separate file) |
| **Assembler** | Translates assembly code to machine code | Machine code |

### Compiler vs Interpreter

| Aspect | Compiler | Interpreter |
|---|---|---|
| Translation | Whole program at once | Line by line |
| Output | Standalone executable | No standalone file |
| Execution Speed | Fast (pre-translated) | Slower (translates during runtime) |
| Debugging | Harder (recompile after changes) | Easier (interactive) |
| Examples | GCC (C/C++), javac (Java) | Python (CPython), Ruby, JavaScript |

---

## Programming Paradigms

| Paradigm | Approach | Languages |
|---|---|---|
| **Procedural** | Step-by-step procedures and functions | FORTRAN, C, BASIC |
| **Logic-oriented** | Rules, facts, and logical inference | Prolog |
| **Object-oriented (OOP)** | Objects (data + methods), encapsulation, inheritance, polymorphism | C++, Java, C#, Python |

---

## Characteristics of a Good Programming Language
- Simple, clear, and concise syntax
- Well-documented function library
- Efficient use of memory and resources
- Integrated Development Environment (IDE) support
- Platform independence
- Object-oriented features (encapsulation, inheritance, polymorphism)
- Consistent syntax and semantics
- Extensibility and maintainability
