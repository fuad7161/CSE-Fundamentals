# Number Systems and Data Representation

How data is represented inside a computer using different number systems and encoding schemes.

### 12.1 — Binary, Octal, Decimal, Hexadecimal — Conversions
- Decimal (Base 10): digits 0–9 (everyday use)
- Binary (Base 2): digits 0,1 (computer internal)
- Octal (Base 8): digits 0–7
- Hexadecimal (Base 16): digits 0–9, A–F
- Conversion methods:
  - Decimal → Binary (repeated division by 2)
  - Binary → Decimal (powers of 2)
  - Binary → Octal (group of 3 bits)
  - Binary → Hex (group of 4 bits)
  - Octal/Hex → Binary (expand each digit)
- Fractional number conversions

### 12.2 — ASCII, Unicode — Basic Concept
- ASCII: 7-bit (128 characters), extended ASCII (8-bit, 256 characters)
- Unicode: universal encoding (UTF-8, UTF-16, UTF-32)
- UTF-8: variable-width, backward-compatible with ASCII, web standard
- Why Unicode: supports all languages and symbols

### 12.3 — Data Representation — Bit, Byte, Word
- Bit: smallest unit (0 or 1)
- Byte: 8 bits (represents one character)
- Word: CPU's natural data size (16-bit, 32-bit, 64-bit)
- Nibble: 4 bits (half a byte)
