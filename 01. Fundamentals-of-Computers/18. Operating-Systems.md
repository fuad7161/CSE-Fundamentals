# Operating Systems

Comprehensive study notes on operating system concepts — process management, memory management, file systems, OS types, user interfaces, and popular operating systems.

---

## Process Management

### Process States
```
New → Ready → Running → Terminated
         ↓          ↓
      Waiting    Blocked
         ↓
      Suspended
```

### Process Control Block (PCB) Contents
- **Process ID** — Unique identifier for the process
- **Process State** — Current state (new, ready, running, etc.)
- **Program Counter** — Address of the next instruction
- **Register Information** — CPU register values when process was last running
- **Scheduling Information** — Priority, queue pointers
- **Memory Information** — Memory allocated to the process
- **I/O Status** — List of open files, allocated I/O devices

### Scheduling Algorithms

| Algorithm | Description |
|---|---|
| **FCFS** (First Come First Served) | Simplest; processes executed in arrival order |
| **SJF** (Shortest Job First) | Process with shortest burst time executed first |
| **Priority Scheduling** | Highest priority process executed first |
| **Round Robin** | Each process gets a fixed time quantum; if not complete, returns to queue |

## Memory Management

| Technique | Description |
|---|---|
| **Segmentation** | Memory divided into variable-sized logical segments (code, data, stack) |
| **Paging** | Memory divided into equal-sized pages; non-contiguous allocation |
| **Swapping** | Moving processes between main memory and secondary storage to free RAM |

## File Management

### File Attributes
- Name, Type, Location, Size
- Date/Time created, modified, accessed
- Read-only, Hidden, Archive flags

### File Operations
- Create, Save, Open, Close
- Modify, Rename, Delete

### File Systems

| System | Characteristics |
|---|---|
| **FAT** (File Allocation Table) | Simple, for smaller disks; compatible across OSes |
| **NTFS** (New Technology File System) | Robust, journaling, security permissions, for larger disks |

## Types of Operating Systems

| Type | Description | Examples |
|---|---|---|
| **Batch Processing** | Jobs processed in batches without user interaction | Early OS |
| **Multi-user** | Multiple users simultaneously (via terminals) | Unix, Linux |
| **Multitasking** | Multiple tasks/programs run simultaneously | Windows, Unix |
| **Real-time (RTOS)** | Time-critical applications; guaranteed response time | MTOS, Lynx, VxWorks |
| **Multiprocessor** | Supports multiple CPUs/cores | Windows, Unix |
| **Embedded** | Runs on dedicated devices with limited resources | Palm OS, Windows CE, Embedded Linux |

---

## User Interfaces

| Interface | Method | Pros | Cons |
|---|---|---|---|
| **GUI** (Graphical User Interface) | Point-and-click, icons, windows | User-friendly, intuitive | Resource-intensive, slower for power users |
| **CLI** (Command Line Interface) | Text-based commands | Efficient, powerful, scriptable | Steep learning curve, requires memorization |

## Popular Operating Systems

### MS-DOS (1981)
- Single-user, single-tasking
- Command line interface
- Key system files:
  - `IO.SYS` — Hardware initialization
  - `MSDOS.SYS` — DOS kernel
  - `CONFIG.SYS` — System configuration
  - `COMMAND.COM` — Command interpreter (shell)
  - `AUTOEXEC.BAT` — Startup batch file

### UNIX
- Multi-user, multitasking
- Core components:
  - **Kernel** — Central core; manages hardware, memory, processes
  - **Shell** — Command interpreter between user and kernel
  - **Files and Processes** — Everything is a file or a process
- Hierarchical directory structure (root: `/`)

### Windows
- GUI-based operating system by Microsoft
- Multi-user, multitasking
- Key versions: Windows 95, 98, 2000, ME, XP, Vista, 7, 8, 10, 11
