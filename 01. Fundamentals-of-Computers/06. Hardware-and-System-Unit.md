# Hardware and System Unit

Detailed coverage of the internal components of a computer — the system unit, CPU architecture, motherboard, and memory hierarchy.

### 2.2 — CPU: ALU, CU, Registers
- ALU (Arithmetic Logic Unit): performs arithmetic & logical operations
- CU (Control Unit): directs data flow, decodes instructions
- Registers: small, high-speed storage inside CPU (accumulator, program counter, instruction register, MAR, MDR)

### 2.3 — Von Neumann Architecture
- Stored-program concept: instructions and data share the same memory space
- Single bus for data and instructions (bottleneck known as Von Neumann bottleneck)
- Harvard Architecture: separate memory for data and instructions

### 2.4 — Machine Cycle
- Fetch: instruction fetched from memory
- Decode: CU decodes the instruction
- Execute: ALU executes the operation
- Store: result written back to memory/register

### 4.1 — Motherboard — Components & Function
- Main PCB connecting all components
- Chipset (Northbridge + Southbridge), CPU socket, RAM slots, expansion slots, I/O ports
- Form factors: ATX, Micro-ATX, Mini-ITX

### 4.2 — Microprocessor — Structure, Clock Speed, Word Length
- CPU cores, clock speed (GHz), word length (8-bit, 16-bit, 32-bit, 64-bit)
- Multi-core processors, hyper-threading

### 4.3 — Primary Memory: RAM & ROM
- RAM: volatile, SRAM vs DRAM (SRAM faster, used for cache; DRAM cheaper, used as main memory)
- ROM: non-volatile, PROM (programmable once), EPROM (erasable by UV), EEPROM (electrically erasable)
- RAM is read/write; ROM is read-only in normal operation

### 4.4 — Cache Memory — Levels
- L1 cache: fastest, smallest (per core)
- L2 cache: medium speed/size (per core or shared)
- L3 cache: shared among cores, slower but larger
- Purpose: bridge speed gap between CPU and main memory

### 4.5 — Expansion Slots & Cards
- PCIe slots: graphics card, sound card, network card
- GPU: dedicated vs integrated graphics
- Add-on cards: Wi-Fi, TV tuner, capture card

## Supplementary Notes from Master Study Guide

### Computer Components Overview

| Component | Description |
|---|---|
| CPU | "Brain" of computer - controls and executes instructions |
| Monitor | Displays information visually |
| Keyboard/Mouse | Input devices |

### CPU Components (Detailed)

| Component | Function |
|---|---|
| Arithmetic Unit (AU) | Performs arithmetic operations (+,-,×,÷) |
| Logic Unit (LU) | Performs logical operations (>,<,=,AND,OR) |
| Control Unit (CU) | Controls flow of data and information |
| Main Memory | RAM - temporary storage |
| Cache Memory | Fast memory for frequently accessed data |
| Registers | High-speed temporary storage |

### CPU Cycle (Machine Cycle) - Summary
```
Fetch → Decode → Execute → Store
```

| Phase | Description |
|---|---|
| Fetch | Retrieves instruction from memory |
| Decode | Breaks instruction into opcode and operands |
| Execute | ALU performs the operation |
| Store | Results stored back in memory |

### Cache Memory Types

| Type | Location | Speed | Size |
|---|---|---|---|
| L1 Cache (Primary) | Inside CPU | Fastest | Smallest |
| L2 Cache (Secondary) | On Motherboard | Slower | Larger |

### Registers Reference

| Register | Purpose |
|---|---|
| Program Counter (PC) | Tracks next instruction |
| Instruction Register (IR) | Holds instruction to be decoded |
| Memory Address Register (MAR) | Holds memory address |
| Memory Buffer Register (MBR) | Stores data from/to CPU |
| Memory Data Register (MDR) | Stores operands and data |
| Accumulator (ACC) | Stores ALU results |

### Processor Communication

**Memory Read Operation:**
1. Processor loads address into MAR
2. Issues READ control signal
3. Memory loads data into MDR
4. Data transferred to processor

**Memory Write Operation:**
1. Processor loads address into MAR
2. Processor loads data into MDR
3. Issues WRITE control signal
4. Memory stores data

### Instruction Sets: CISC vs RISC

| Feature | CISC | RISC |
|---|---|---|
| Instructions | 100-250 complex | 0-100 simple |
| Format | Variable length | Fixed 32-bit |
| Speed | Slower | Faster (single clock cycle) |
| Design | Complex hardware | Simple hardware |
| Memory Access | Frequent | Minimal (register-based) |
