# Traditional Data Center vs. Cloud Data Center

Comparison between on-premises infrastructure and cloud-based data center models.

- **Traditional (On-Premises) Data Center**
  - Organization owns and manages all hardware and software
  - High upfront capital expenditure (CapEx)
  - Fixed capacity — must over-provision for peak loads
  - Maintenance, security, and upgrades handled internally
- **Cloud Data Center**
  - Infrastructure owned and managed by cloud provider (AWS, Azure, GCP)
  - Pay-as-you-go operating expenditure (OpEx) model
  - Elastic scaling — resources adjust dynamically to demand
  - Provider handles maintenance, patching, and physical security
- **Key differences**: ownership, cost model, scalability, management overhead, SLA guarantees
