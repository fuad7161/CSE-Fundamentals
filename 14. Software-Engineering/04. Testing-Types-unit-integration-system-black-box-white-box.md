# Testing Types — unit, integration, system, black box, white box

Classification of software testing strategies across levels and approaches.

- **By level**:
  - **Unit testing**: testing individual components/functions in isolation
  - **Integration testing**: verifying interactions between modules
  - **System testing**: end-to-end testing of the complete system
  - **Acceptance testing**: validating against user requirements
- **By approach**:
  - **Black-box testing**: testing functionality without knowledge of internal structure
  - **White-box testing**: testing internal logic, paths, and conditions
  - **Gray-box testing**: combination of both
- Test automation and test case design techniques
- Regression testing, smoke testing, and sanity testing
