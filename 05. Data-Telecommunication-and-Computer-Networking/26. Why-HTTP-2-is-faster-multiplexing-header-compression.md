# Why HTTP/2 Is Faster — Multiplexing & Header Compression

Key optimizations that make HTTP/2 outperform HTTP/1.1. Covers:

- **Multiplexing** — multiple streams over a single TCP connection, eliminating head-of-line blocking
- **Header compression (HPACK)** — reduces redundant header data, cutting overhead
- **Server push** — proactively sends resources before client requests them
- **Binary framing** — more efficient parsing than text-based HTTP/1.1
